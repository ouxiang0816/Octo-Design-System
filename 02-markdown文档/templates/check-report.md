# Template：设计检查结果页

适用场景：设计规范检查、组件一致性检查、无障碍 / 颜色 / 字体 / 间距 / 命名检查、质量验收报告。

## 首屏结构（必须包含）

1. **顶部摘要**：检查对象、检查时间、通过率、问题总数（使用紧凑指标块，不堆大卡片）
2. **分类筛选**：颜色 / 字体 / 间距 / 组件 / 可访问性 / 命名等分类 Tab 或 Filter
3. **问题列表**：严重级别 + 问题描述 + 所在页面或节点 + 修复建议
4. **右侧详情面板**：问题截图、定位信息、建议 token、修复操作
5. **批量操作**：导出报告 + 应用修复 + 重新检查

## 布局尺寸

| 项目 | 值 |
|---|---|
| 摘要指标块 | 紧凑横排，不用大卡片堆叠 |
| 问题列表行高 | 44–52px |
| 详情面板宽度 | 360–420px |
| 严重级别颜色 | 只用于状态点、Badge 或小面积文本，不铺大背景 |

## 必备组件

`Badge` · `Table` · `Tabs` · `Progress` · `Alert` · `Sheet` · `Button` · `DropdownMenu`

## 禁止项

- 不只生成图表而缺少可执行问题列表
- 不用红色铺满错误区域
- 不只展示总分，必须能定位到具体问题
